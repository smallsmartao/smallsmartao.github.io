强化学习在 LLM-based ASR 和 TTS 系统中的应用探索

强化学习在 LLM-based ASR 和 TTS 系统中的应用探索

论文信息

  • 标题: Explore the Reinforcement Learning for the LLM based ASR and TTS system
  • 链接: https://arxiv.org/abs/2509.18569

一、背景与动机

1.1 LLM 在语音领域的应用

近年来，大语言模型（LLM）在语音领域取得了显著进展，包括：

  • ASR（自动语音识别）：将语音转换为文本
  • TTS（文本转语音）：将文本转换为语音

目前大多数 ASR 和 TTS 系统都基于 LLM，相较于传统小模型有显著优势。

1.2 强化学习在 LLM 中的作用

在 LLM 训练中，强化学习（RL）是关键步骤：

  • 使模型与人类偏好对齐
  • 增强模型的推理能力

常见的 RL 方法包括：

  • PPO（Proximal Policy Optimization）
  • GRPO（Group Relative Policy Optimization）
  • DiffRO（Differentiable Reward Optimization）

1.3 本文研究问题

相较于文本和视频模态，音频 LLM 的 RL 研究仍然有限。本文详细研究了 RL 在 LLM-based ASR 和 TTS 系统中的效果。

二、方法详解

2.1 音频 LLM 的 RL 训练框架

本文设计了一个交替 GPU 利用框架，高效管理音频 LLM 的 RL 训练：

ASR 训练流程：

  1. PyTorch 音频编码器占用 GPU，提取音频 embedding
  2. SGLang LLM 生成多组假设，计算奖励
  3. FSDP LLM 策略模型执行策略优化

TTS 训练流程：

  1. SGLang 生成 acoustic token 序列
  2. Flow-matching 模型和 vocoder 生成波形
  3. 奖励模型计算奖励分数
  4. FSDP 策略模型执行 GRPO 或 DiffRO

2.2 GRPO 在 ASR 中的应用

奖励函数设计：

奖励 描述
$R^1$ ASR 准确率：$1 - WER(y^*, y)$
$R^2$ 幻觉检测：检测重复、不存在、翻译结果
$R^3$ 关键词准确率和召回率

RL 训练数据构建：

  • $D^1$：不同 ASR 系统转录差异大的样本（hard samples）
  • $D^2$：长音频（>20秒）
  • $D^3$：包含专有名词（品牌、人名）的样本

2.3 DiffRO 与 GRPO 结合用于 TTS

DiffRO 原理：

  • 使用 GumbelSoftmax 从 LLM 采样输出 token
  • 用 token-based ASR 计算奖励
  • 梯度可以直接反向传播到 TTS 模型

GRPO 奖励设计：

奖励 描述
$R^1$ ASR 识别准确率
$R^2$ 音频时长：防止语音过慢
$R^3$ Token 和 Pitch 多样性

结合方法（Sample Filter）：

  • 根据 advantage 区分正负样本
  • 只对正样本计算 DiffRO loss
  • 使训练更稳定有效

三、实验结果

3.1 训练效率

  • ASR：1小时音频数据，RTF ≈ 0.015
  • TTS：单步训练耗时 16.73s（batch=128）

3.2 ASR 实验结果

配置 短语音 WER 长语音 WER
Baseline - -
GRPO (全部) 5.3% 相对提升 5.3% 相对提升

关键发现：

  • 幻觉问题在长音频中更严重
  • 关键词相关数据和奖励设计对缓解幻觉至关重要
  • Hard samples ($D^1$) 对长语音提升明显

3.3 TTS 实验结果

方法 WER 说话人相似度 (SS)
Baseline - -
DiffRO 更优 下降较多
GRPO 改善 轻微下降
Sample Filter 结合 最佳 最佳平衡

关键发现：

  • DiffRO 在 WER 上更好，但 SS 下降更多
  • Sample Filter 结合方法效果最佳
  • 音频时长奖励可以防止语音速度变慢

四、总结

本文探索了 RL 在音频 LLM（ASR 和 TTS）中的应用：

  1. 提出交替 GPU 利用框架，训练效率高
  2. ASR：GRPO + 多种奖励函数，WER 相对提升 5.3%
  3. TTS：结合 DiffRO 和 GRPO，通过 Sample Filter 实现最佳效果
  4. 奖励设计对塑造模型输出偏好和用户体验至关重要

局限与未来方向：

  • 需要更高效的音频 RL 框架
  • 探索更多奖励函数设计
