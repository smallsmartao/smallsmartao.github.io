一、背景与动机

1.1 Speech-aware Language Models (SLMs)

语音感知语言模型（SLM）是目前开源 ASR 排行榜上表现最好的模型架构。它们通常由以下组件构成：

  • Conformer 声学编码器：使用 CTC loss 或 RNN-T loss 训练
  • 语音模态适配器：MLP 或 Query Transformer，将声学嵌入投影到 LLM 可理解的空间
  • 大语言模型：负责生成文本输出

1.2 核心问题

自回归（AR）推理是 SLM 的主要瓶颈：

  • 每个 token 都需要一次 LLM 前向传播
  • 无法像 CTC 那样并行解码
  • 延迟高，不适合实时场景

1.3 现有加速方法

Speculative Decoding：

  • 使用小型 draft 模型并行生成候选 token
  • 大模型在单次 forward 中验证候选
  • 验证失败则回退到 AR 解码

本文创新：直接复用 SLM 的 CTC 编码器作为 draft model，无需额外训练。

2.2 关键技术

1. 帧级熵阈值 (Frame Entropy Threshold)

使用 CTC 输出分布的帧级熵作为置信度指标：

$$\sum_{a \in \bar{Y}} -p(a|h_t) \log p(a|h_t) < \tau_{CTC}$$

熵低说明模型对当前帧的预测很确定，可以直接接受。

2. Token 似然验证 (LLM Verification)

用 LLM 验证 CTC hypothesis：

$$p(y_i|\hat{y}{<i}, g) > \tau{SLM}$$

由于 LLM 是因果模型，所有 token 的验证可以在一次前向传播中并行完成。

3. 宽松接受准则 (Relaxed Acceptance Criterion)

验证失败时，不需要完全匹配，只需找到最长已验证前缀，然后从该前缀继续 AR 解码。这比传统的 speculative decoding 更灵活。

3.3 主实验结果

在 HuggingFace Open ASR 基准上，使用 1B 参数 LLM + 440M 参数 CTC 编码器：

模式 WER RTFx (逆实时因子) 相对 AR WER 变化
Full AR 5.75% 564 -
High Accuracy (SSD) 5.58% 548 -3% (提升)
High RTFx (SSD) 6.56% 2491 +12%

关键发现：

  • 高精度模式：同时提升准确率（WER 降低）和吞吐量
  • 高吞吐模式：4.4x 加速，WER 仅增加 12%

3.4 消融实验

验证两个验证阶段是否都必要：

  • 只用 CTC 接受：高吞吐但无法达到最低 WER
  • 只用 LLM 验证：高准确但 RTFx 较低
  • 两者结合：Pareto 最优

3.6 关键洞察

LLM 验证能够降低 WER，是因为 CTC 和 SLM 产生互补的错误模式。

AR 输出通常更流畅但不一定忠实于音频（LM bias 问题），而 CTC 验证可以纠正这些错误。

四、模型细节

  • CTC 编码器：440M 参数，16 层 Conformer
  • Projector：37M 参数，2 层 Query Transformer
  • LLM：Granite-4.0-nano (1B 参数)
  • LoRA：rank=64
  • 训练数据：90,000 小时多语言数据

五、总结

本文提出了一种创新的自投机解码方法：

  1. 复用 CTC 编码器作为 draft model，无需额外训练
  2. 双阶段验证：帧熵 + LLM 似然，平衡速度与精度
  3. 精度与速度兼得：高精度模式 WER 比纯 AR 还低

局限：

  • 需要带 CTC 头的 SLM
  • 验证失败时需要整句回退 AR 解码

未来方向：

  • 联合训练编码器与 LLM，专门优化投机解码
  • 应用于实时对话场景
