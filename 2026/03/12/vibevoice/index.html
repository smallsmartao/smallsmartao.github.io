VIBEVOICE-ASR 论文解读

VIBEVOICE-ASR 论文解读

背景与动机

长音频（会议、播客等）的语音识别面临两个核心挑战：

  1. 上下文碎片化 (Context Fragmentation)：长音频被切分成多个片段后，各片段独立识别，导致上下文信息丢失
  2. 多说话人复杂性 (Multi-Speaker Complexity)：多人对话场景下的说话人分离和识别难度增加

传统方案通常采用 pipeline 方式：先做说话人分离 (Speaker Diarization)，再切片送 ASR，最后拼接。这种方式存在误差传播问题。

核心贡献

VibeVoice-ASR 提出了一个统一框架，在单一端到端生成模型中完成以下任务：

任务 描述
ASR 自动语音识别
Speaker Diarization 说话人分离
Timestamping 时间戳生成

关键特性

  1. 单次处理能力：支持最长 60 分钟音频的单次处理，无需切片
  2. 多语言支持：支持 50+ 语言，无需显式语言设置
  3. Code-switching 原生支持：原生支持 utterance 内部和跨 utterance 的代码切换
  4. Prompt-based Context Injection：用户可通过 prompt 提供自定义上下文信息，提升领域术语和多音字消歧的准确率

技术方案

统一建模框架

将 ASR、Speaker Diarization、Timestamping 统一为单一生成任务，通过一个模型输出多任务结果，避免 pipeline 方式带来的误差传播。

Context Injection 机制

通过 prompt 方式注入上下文信息，包括：

  • 领域术语（提升垂直领域识别准确率）
  • 多音字消歧（提升专有名词识别准确率）

总结

VibeVoice-ASR 通过统一的端到端框架解决了长音频语音识别的核心痛点，单次处理 60 分钟音频的能力在实际场景中有很大实用价值。其 prompt-based context injection 机制与当前 LLM 趋势结合，是值得关注的亮点。
