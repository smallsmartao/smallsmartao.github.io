LLM based ASR中Hot Word Bias的应用思考

Context Bias是提升特定词汇识别准确率的关键技术，在垂直领域ASR落地中不可或缺。随着大语言模型（LLM）与自动语音识别（ASR）的深度融合，Bias任务的实现逻辑与效果边界均发生了显著变化。本文将从传统方法回顾、LLM带来的核心增益，以及当前LLM-ASR领域的相关Bias工作三个维度展开总结。

一、回顾：LLM based ASR之前的Hot Word Bias方法

在LLM融入ASR体系之前，行业内已形成两类核心的Hot Word Bias实现方案：一类是基于WFST的传统解码层加权方法（如On-the-fly Rescoring），另一类是更灵活的基于神经网络的Bias方法（NN-Bias，含C-LAS、Context Transformer Transducer等代表性方案）。关于这些传统方法的详细技术细节、实验结果及代码实现，可参考我此前的博客《NN-Bias总结》：NN-Bias总结。

整体来看，传统方法虽能提升热词识别率，但存在明显局限：未登录热词适配成本高、上下文语义建模能力弱、易出现偏置过度导致误召回等问题。而LLM的融入，恰好为解决这些痛点提供了新的思路。

二、LLM为Hot Word Bias任务带来的核心增益

LLM的加入对于ASR中的热词的影响有两个主要方面，分别是模型能力和工具方法，具体体现在以下三点，其中前两点是我说的能力，最后则是一些可以利用的工具和方法：

1. 更强的语言先验知识覆盖，降低未登录热词识别门槛

传统ASR的语言模型（如N-gram、基础Transformer LM）的文本语料覆盖范围有限，多依赖于ASR专属的音频-文本平行语料，导致大量垂直领域热词（如医疗领域的“硝苯地平”、金融领域的“北向资金”）成为未登录词，需额外进行词表扩充与适配。而LLM的训练语料覆盖海量通用文本与各类专业文本，其词汇覆盖率远超传统ASR的LM——即使是部分生僻热词，LLM也大概率已在训练过程中习得其语义与拼写形式。

这种优势让Context Bias任务无需过度依赖“热词词表扩充+微调”的模式：对于传统ASR完全陌生的热词，LLM可凭借自身的语言先验知识，直接理解其语义属性与上下文适配场景，通过偏置引导即可实现精准识别。

2. 更强的上下文利用能力，提升热词识别的精准度

传统ASR解码器的上下文利用能力较弱，其效果接近于n-gram模型，因此本质上不具备基于语义判断词汇合理性的能力。例如在句子“患者服用硝苯地平后血压下降”中，传统ASR可能因“硝苯地平”发音复杂而误判为“心痛地平”，且无法借助“患者”“血压”等上下文信息修正这一错误。

而LLM具备强大的上下文理解与推理能力，其建模的上下文窗口可覆盖数百甚至数千个token。在Bias任务中，LLM不仅能接收“热词列表”的引导，还能结合当前语音对应的文本上下文，判断热词是否符合语义逻辑——即便热词发音存在模糊性，也能通过上下文推理排除错误选项，同时避免非热词的误召回（如不会将普通词汇错误识别为热词）。反观WFST或早期的NN-Bias方法，其核心逻辑都是提升近似发音词汇的发射概率，本质上不具备上下文判断能力，因此工程实践中常见的优化手段是在WFST中加入前导词约束，以减少过度偏置带来的误报问题。

3. 适配热词任务的技术工具迁移，提升方案灵活性

LLM生态中已发展出多种成熟的“外部信息融合”与“强化学习”技术，这些技术可直接迁移至Hot Word Bias任务，大幅提升方案的适配性与效果可控性。其中最具代表性的两类技术的应用逻辑如下：

  • RAG：热词词表本质是一类结构化外部信息，传统方法需将热词硬编码到解码规则或词表中，不仅灵活性差，若要支持大词表还需投入大量优化工作。而基于RAG的思路，可将热词词表构建为检索库，在LLM解码过程中实时匹配上下文并调用热词信息，大幅提升适配效率。

  • RLHF/GRPO：辅助loss曾是NN-Bias领域的研究热点，核心原因在于ASR的基础loss未针对热词任务优化，因此需额外增设辅助loss提升效果。而在LLM框架下，热词任务天然适配GRPO等强化学习方法：可直接将“热词正确召回”“无过度偏置误报”作为正向奖励信号，引导模型自主学习最优偏置策略。

三、LLM based ASR领域的相关Bias工作现状

上文提到，LLM为Context Bias任务带来了新的可能性，本节将结合最新论文深入介绍这一方向的研究进展。

3.1 核心挑战：大词表热词偏置

虽然 LLM-ASR 在各种任务上取得了显著进展，但大词表下的实体和热词上下文偏置仍然是一个核心挑战。这些词汇（如联系人列表、地点等）通常具有高语义或商业价值，但：

  • 在训练数据中出现频率低
  • 可能是未登录词（OOV）

3.2 两阶段框架：热词检索 + ASR 适配

最新研究（arXiv:2512.21828）提出了一种可扩展的两阶段框架，将上下文偏置分解为两个紧密耦合的子问题：

阶段一：基于 GLCLAP 的热词检索

Global-Local Contrastive Language-Audio Pre-trained Model (GLCLAP) 的工作流程：

  1. 音频编码器提取音频 embedding
  2. 文本编码器将候选偏置词编码为语义向量
  3. 计算相似度，选择 top-k 候选热词

三大增强策略：

  1. Robustness-Aware Data Augmentation (RADA)

    • 目的：减少热词词表大小（词表越大，recall 越低，干扰越多）
    • 方法：用 TTS 合成热词语音，用 ASR 解码检查是否能识别
    • 效果：词表从 60万条目缩减到 9.8万

  2. Fuzzy Matching（模糊匹配）

    • 问题：传统方法要求精确匹配，但实际应用中用户可能使用变形、部分提及或释义
    • 方法：在训练数据中加入上下文句子和故意扰动的偏置词变体
    • 效果：显著提升 recall 和鲁棒性

  3. RL 引导的判别训练

    • 问题：需要抑制误报（false positives）
    • 方法：使用 GRPO 强化学习，奖励函数设计：
      • Match reward：候选词同时出现在输出和参考中，或都不出现 → 奖励 1
      • WER-based reward：$1 - WER$，确保转写准确率

阶段二：热词感知的 ASR 适配

  1. 构建偏置提示：

    • 提示中故意包含不相关的干扰词，抑制过度依赖

  2. 联合 Beam Search：在推理阶段联合解码上下文无关和上下文条件假设

3.3 实验结果

热词检索性能

方法 Recall@10 (Medical) Recall@10 (Media)
GLCLAP 42.37% 43.58%
+RADA 67.56% 76.61%
+Fuzzy Matching 85.12% 93.58%

ASR 性能

配置 Media KER Media SACC Medical KER General SACC
Base 22.63% 72.99% 23.31% 77.49%
+GLCLAP (top-2) 11.99% 80.57% 15.89% 77.36%
+GRPO 8.29% 85.78% 11.24% 78.16%

关键发现：

  • KER 大幅降低：Media 从 22.63% 降到 8.29%（相对降低 63%）
  • SACC 在 General 任务上也提升了：说明 GRPO 的 WER 奖励有效
  • top-k 并非越大越好：过大的 k 会引入更多干扰词，反而降低效果

3.4 小结

这一工作完美体现了前文提到的三个核心增益：

  1. 更强语言先验：LLM 能理解未登录热词
  2. 更强上下文利用：通过结构化提示和 GRPO 学习正确使用热词
  3. RL 技术迁移：GRPO 奖励函数同时优化热词召回和整体准确率
